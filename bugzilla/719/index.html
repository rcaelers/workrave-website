Comment 3
Rob Caelers  Apr 13 2008 18:30:18 UTC
It seems that "suspend timer when inactive : disabled" doesn't really work well for the daily limit if you continue working after the daily limit was reached. This is what happens:

- You start working, the daily limit timer starts
- 8 hours later Workrave informs you that you reached the daily limit and stops the daily limit timer.
- You ignore the daily limit: the timer starts running again
- 1 hour later you stop working, the timer keeps on running.
- At 4:00, Workrave resets the daily limit, the timer continues running
- At 7:00, you start working again, the timer is still running, 3 hours of your daily limit time has passed.

I changed workrave to halt the daily limit timer when it resets at 4:00, so it should start at 0:00 when you start working in the morning.

I'm closing this bug because I expect this fix will solve your problem and also removes the need for a manual reset menu option. I'd rather not add such a feature unless necessary. Feel free to let us know I you don't agree.

I've started a new 'nightly' build that contains the fix. A new build should be available for download in an hour.
