Ray Satiro  Sep 6 2012 07:12:11 UTC
Try the debug version of that snapshot:
http://snapshots.workrave.org/20120211/workrave-win32-v1_9_904-17-g12247837ff-dirty-20120211-debug-installer.exe

Create a directory, C:\temp, if you don't already have that directory.

Workrave will start and a command window will open. It will write a debug log to C:\temp. You may notice Workrave is slower. It's constantly writing information to its debug log.

The next time your audio goes out note the important times at which you noticed there is a problem and then exit workrave. Go to the C:\temp directory and look for the most recent workrave-date-time file. Zip up the file and upload it here.

The log files created by the debug version of Workrave are large. It takes about 3 minutes to write a megabyte of data regardless of if you're using your computer. If the log file is becoming too large consider exiting workrave, deleting the log and starting again.
