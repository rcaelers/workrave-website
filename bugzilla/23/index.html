Raymond Penners  Oct 22 2002 01:53:19 UTC
The native help mechanisms are not the problem here. The bulk of the work is
writing the documentation itself. And, if the documentation is written in
docbook it can be easily converted to HTML and other (online help) formats. 

So, regardless of what type of documentation your are going to write, be it for
the web or whatever, if you do so in docbook, adding online help using native
mechanisms is relatively easy to do.

Oh, I noticed your set the component from Core to Gui-Gtkmm. Both are wrong
actually, we need a "Gui" component.
