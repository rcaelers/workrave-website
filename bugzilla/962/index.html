Rob Caelers  Jan 12 2011 18:57:21 UTC
I assume you are referring to my advice in http://issues.workrave.org/show_bug.cgi?id=942#c7. That was about a week before 1.9.3 was released, so it's unlikely that you downgraded from 1.9.3 to 1.9.2-62 :-) You probably upgraded from 1.9.2 to 1.9.2-62. 

The difference between 1.9.2-62 and 1.9.3 is minimal (and limited to the Linux version of Workrave). Many users have reported that 1.9.3 fixes the monitoring problems. So installing http://snapshots.workrave.org/20110111/workrave-win32-v1_9_3-16-gbfb2206b99-20110111-installer.exe should be safe :-)

About the 'Do you want to allow this program to make changes to this computer' and startup delays: I have no idea what is causing this; it's not a message  Workrave is displaying. Perhaps some virus protection/malware protection program is blocking Workrave. If Workrave starts successfully and you can access the preferences: try to disable networking.

In bug 942 you mentioned that 1.9.2-62 works correctly. So I assume that the problems you describe in this bug report started at a later time. Did anything change on your computer before Workrave started to misbehave? (for example a new virus scanner)
