Comment 4
Rob Caelers  Oct 24 2011 21:54:59 UTC
Hi, I haven't been able to reproduce this bug on Windows 7 PC yet. I will try the 'hide hide status window' trick as soon as I have access to a windows 7 machine again. 

The "quiet mode" is intentional: Workrave's preferences are instant-apply; if you change the 'time between breaks', the new value is used immediately, In normal mode, this could result in a break window without skip/postpone buttons... For this reason, Workrave enters 'quiet' mode as long as the preferences windows is focused.
Comment 5
an**@ya**.com  Oct 26 2011 03:01:26 UTC
Yeah, too bad, I've had to stop using WR till this is fixed.

In case it matters, here's what I do when I install it:

1. Close the little timer window and open Preferences.

2. In the Timers group:

  a. Disable Micro Break and Daily Limit features.

  b. On the Rest Break tab, leave all defaults except:

     _ Time between breaks: 30 min.

     _ Break duration: 5 min.

3. In the User Interface group:

  a. Sounds tab: No sounds

  b. Status Window tab: Clear "Show status window" checkbox

  c. Applet tab: Clear "Show applet" checkbox (I don't even know what this is, BTW)
