Jonathan Van Dusen  Sep 22 2013 18:51:29 UTC
I haven't had issues with using Dexpot and Workrave together on Windows 8, as I can be using a secondary desktop and still receive Workrave prompts. However, I'm having other problems with Workrave itself on Windows 8, which might be relevant to the Dexpot issue:

1) When I'm using Windows Store ("Metro-style") apps in Windows 8, Workrave may still launch a micro-break or rest-break prompt on the Desktop, in which case the prompts are not visible to me. This might be a product of the sandboxing between Desktop programs and Metro-style apps in Windows 8, whereby Workrave might not be able to tell that a Metro-style app currently has focus. But would it be possible for Workrave to force Windows 8 to switch back to the Desktop whenever one of these prompts is created? Better yet, could Workrave use Windows 8's full-screen notifications (as seen for low-battery notifications, for example) to display the micro-break and rest-break prompts?

wa**@ya**.com, this may be the issue you're experiencing -- if you're using Windows 8 with Metro-style apps, and aren't use the Desktop or Dexpot at that moment, you might not be seeing the Workrave prompts. Or did your issue go away when you uninstalled Dexpot?

2) Similar to the above problem, when I use the "Block input" or "Block input and screen" functions, I am unable to use some of the Metro functions when a Workrave prompt appears on the Desktop underneath. For example, in this scenario, the Start screen is unresponsive to mouse and keyboard input, and mousing to the corners of the screen (for Windows 8's "Switcher" and "Charms" interfaces) also does not work when using the Start screen or any Metro-style app. (Touch input still seems to work correctly.) Could there be an option added to block the screen but not the input?

Would be happy to provide more information about these scenarios as needed. I've appreciated using Workrave.
