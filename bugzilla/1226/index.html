Description
fl**@be**.biz  Nov 24 2015 21:46:13 UTC
I am running a Synergy server and Workrave on my Windows desktop. If a break occurs while my mouse focus is on my laptop, Workrave's break window starts flashing orange, indicating (I think) that it's detecting activity on the mouse/keyboard and the break timer stops indefinitely, even if I move the mouse focus back to my desktop.

In this case I also have Workrave set to block the screen and input, so it's not as if I can actually be active while the break is taking place, so it would be nice if in this situation Workrave didn't try to see if I was typing through my break. Or, more generally, being able to turn off the behavior where Workrave extends the break if the user keeps on working through it.
Comment 1
Rob Caelers  Dec 23 2015 16:36:07 UTC
If I understand correctly, Synergy synchronizes keyboard and mouse between computers. So if you get a break on your laptop and then continue workrave on your desktop, Workrave will still see activity on your laptop. So I assumes you are ignoring the break.

Did you try the alternative activity monitor setting in the preferences? Perhaps this changes the interaction between Workrave and Synergy.
Comment 2
fl**@be**.biz  Jan 5 2016 21:58:51 UTC
Hi, sorry for my delay in response, but I was on vacation for the holidays.

I am only running Workrave on my desktop; my laptop is a Mac, and I just take advantage of the fact that Workrave still sees the keyboard and mouse activity that Synergy is forwarding along to the Mac. That is, while I'm working on the laptop, the breaks still occur on the desktop, which is as intended; the problem I'm running into is that if the break occurs while the laptop has focus, the break never ends because the desktop continues to see activity during the break, even if I move the focus back to the desktop during the break.

I will try the "use alternate monitor" setting as you suggested, and see if that helps with this particular issue.
