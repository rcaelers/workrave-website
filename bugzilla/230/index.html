Kees-Jan send me this via icq:

kees-jan$ grep -i workrave ~/.gnome2/session
4,Program=workrave
4,CloneCommand=src/workrave/src/workrave --sm-config-prefix /workrave-ddd5ku/
--display :0.0
4,RestartCommand=src/workrave/src/workrave --sm-config-prefix /workrave-ddd5ku/
--sm-client-id 11c0a8007f000104999128600000004020003 --screen 0 --display :0.0

If "src/workrave/src/workrave" cannot be found in your path, workrave is not
started by the session manager.

Workrave now stores the full pathname to the workrave binary in the session. I
Hopefully this fixes the problem.
